Embed and play videos easily in the Gutenberg editor with a customizable and responsive Video Player Block for WordPress.

== Description ==

Video Player Block is a powerful and versatile WordPress plugin that lets you embed and play videos with style. Beyond the standard HTML5 player, it offers support for industry-leading engines like **React Video Player**, **Video.js**, and **Vidstack**, making it the ultimate tool for professional video sites.

Whether you are showcasing YouTube, Vimeo, self-hosted MP4s, HLS streams, or external media from services like Amazon S3 or BunnyCDN, this plugin provides a clean, responsive, and highly customizable video experience directly within the Gutenberg editor.

**[Demos](https://bblockswp.com/demo/video-player-block-all-demos/)**

=== Key Features – Free Version ===
- **Default HTML5 Video Player:** A clean, lightweight, and responsive player for standard self-hosted MP4, WebM, and OGG formats.

- **YouTube & Vimeo Native Support:** Easily embed and play YouTube and Vimeo videos within the customizable player skin.

- **Native Gutenberg Block:** Add videos directly from the WordPress editor with a dedicated, easy-to-use block.

- **Fully Responsive Design:** Automatically adjusts the video player for seamless viewing on desktops, tablets, and mobile devices.

- **Customizable Player Controls:** Easily show or hide basic player controls like play, volume, and fullscreen.

- **Rewind & Fast-Forward Buttons:** Toggle dedicated seek backward and seek forward buttons on the player.

- **Player Restart Button:** Add a dedicated button to easily restart the video from the beginning.

- **Download Button Toggle:** Easily enable or disable the video download option for your viewers.

- **Picture-in-Picture & AirPlay:** Toggle PiP and AirPlay options directly from the settings panel.

- **Shortcode Generator:** Generate and insert video players anywhere using shortcodes for maximum flexibility.

- **Poster Image Selection:** Choose and display custom video poster images directly from the WordPress media library.

- **Autoplay & Mute Options:** Control video behavior on page load with simple toggle settings.

- **Lightweight & Fast:** Built for performance to ensure your pages stay fast and user-friendly.

- **Unlimited Video Players:** Add multiple video players anywhere on your website without any restrictions.

- **Cross-Browser & iOS Friendly:** Fully compatible with Chrome, Firefox, Safari, Edge, and mobile devices (iPhone/Android).

- **Google Cast & Chromecast:** Wirelessly stream video content from the browser player to any Chromecast-enabled TV.

- **Video Gallery Block:** Showcase a whole collection of videos in grid, masonry, carousel, or bento layouts with category filters, lazy-loaded thumbnails, and an integrated lightbox or featured playlist mode.

- **Video Hero Block:** Full-bleed landing-page hero with an autoplay muted background video, layered headline / subheadline / CTA buttons, gradient & noise overlays, an optional click-to-play trailer modal, and built-in performance fallbacks for mobile, Save-Data, and prefers-reduced-motion viewers.

- **Video Lightbox Block:** Click-to-play trigger (thumbnail, button, or play-icon overlay) that opens any supported video inside a responsive modal lightbox. Defers all heavy player assets until the visitor opts in, with per-instance engine choice (Plyr / Video.js / Vidstack / native iframe), URL-hash deep links, auto-open-after-delay, exit-intent, and cookie-gated "only once per visitor" auto-open.

- **Video Comparison Block:** Draggable before/after slider that synchronously plays two self-hosted videos side-by-side (horizontal or vertical reveal). Frame-synced dual playback, autoplay-on-scroll, configurable divider / handle / labels, optional unified play & mute controls, and full keyboard accessibility on the divider slider.

- **Video Testimonial Card Block:** Conversion-focused social-proof card pairing a customer photo, attributed quote, half-step star rating, and an inline-playable video clip. Four layout variants (media-left / right / top / background overlay), three play modes (inline / lightbox / hover-preview), self-hosted + YouTube + Vimeo + direct URL sources, three quote treatments, optional "Verified" badge, company logo, and built-in schema.org/Review JSON-LD output for rich-result eligibility.

- **Video Playlist Block:** Side-by-side video player and clickable episode queue with Netflix/YouTube-style sequential viewing. Per-playlist engine choice (Plyr / Video.js / Vidstack / React), mixed sources in a single queue (YouTube, Vimeo, MP4, HLS, DASH, Mux), inline chapter authoring with timeline markers, auto-advance with countdown, loop / shuffle, watched-state checkmarks and resume-where-you-left-off via localStorage, URL-hash deep links (`#video=3&t=125`), four layouts (queue-right / queue-left / queue-below / theater), per-item search filter, and built-in ItemList + VideoObject JSON-LD output for SEO.

- **Video Reels Block:** A vertical, swipe-driven short-form video feed (TikTok / Instagram Reels / YouTube Shorts style) embeddable inside any post, page, or template. IntersectionObserver autoplay-on-visible, mute-by-default with tap-to-unmute, CSS scroll-snap vertical paging, touch swipe + keyboard arrow navigation, mixed MP4 / YouTube Shorts / Vimeo / HLS / DASH sources in a single queue, three layouts (feed-fullscreen / inline-strip / grid-then-fullscreen), per-clip CTA / caption / author pill / hashtags / progress bar / right-rail Like (localStorage) / Share / Mute / Fullscreen, configurable preload strategy, "Load more" pagination, URL-hash deep links (`#reel=3`), and built-in VideoObject + ItemList JSON-LD output for SEO.

- **Interactive Video Block:** A single video enriched with timeline-triggered overlays — CTA buttons, text annotations, image cards, clickable hotspots — that appear at editor-defined start/end timestamps to drive engagement and conversions. Supports self-hosted MP4 / WebM, YouTube, and Vimeo sources with configurable position, color, and CTA style per overlay. Includes inline chapter markers (ticks on the timeline + optional chapter list with click-to-seek), an end-screen with CTA and rewatch button, and an email gate (none / Mailchimp / webhook) that captures the visitor's email before continuing playback. Click and gate events are emitted via `wp.hooks` and a window CustomEvent so GA, Plausible, and custom listeners can record them.

- **Sticky Video Player Block:** A video that pins itself to a screen corner and minimizes on scroll, keeping playback visible while visitors keep reading. Scroll-distance docking trigger, four configurable dock corners with collision-safe margins, "only when playing" gating, minimize-to-tab and close controls with session-persistent dismissal, native Picture-in-Picture button, auto-PiP when the visitor switches tabs, mobile enable/disable, prefers-reduced-motion respect, and slide / fade / none dock animations. Accepts self-hosted MP4, HLS, YouTube, and Vimeo sources.

- **Video Transcript Block:** A synchronized, clickable transcript displayed alongside any self-hosted, YouTube, or Vimeo video. Cue lines highlight as the video plays and clicking a cue seeks the video to that moment instantly. Includes a real-time search/filter input, auto-scroll to keep the active cue in view, optional MM:SS timestamp labels, full color and typography controls, three layout modes (side-by-side / transcript-left / stacked), configurable column split ratios, and automatic VideoObject JSON-LD injection so Google and AI answer engines can index your spoken content directly from the page.

== Build Process ==

This plugin uses a build process to generate the production assets (JS/CSS) located in the `build/` directory. The human-readable source code is available in the `src/` directory.

To build the plugin from source:
1. Clone the repository: https://github.com/bPlugins/video-player-block
2. Install dependencies: `npm install`
3. Run the build command: `npm run build`

Build Tools Used: Webpack, Babel, PostCSS.


== Installation ==

= From Gutenberg Editor: =
1. Go to the WordPress Block/Gutenberg Editor
2. Search For **Video Player Block**
3. Click on the **Video Player Block** to add the block

= Download & Upload: =
1. Download the **Video Player Block** plugin (*.zip file*)
2. In your admin area, go to the Plugins menu and click on **Add New**
3. Click on **Upload Plugin** and choose the **`video-player-block.zip`** file and click on **Install Now**
4. Activate the plugin and Enjoy!

= Manually: =
1. Download and upload the **Video Player** plugin to the **`/wp-content/plugins/`** directory
2. Activate the plugin through the Plugins menu in WordPress
